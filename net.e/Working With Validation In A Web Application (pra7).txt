========================================================
  WORKING WITH VALIDATION IN AN ASP.NET MVC WEB APPLICATION
  A Beginner-Friendly Step-by-Step Guide
========================================================

WHAT IS FORM VALIDATION?
--------------------------
When a user fills out a form on a website, validation is the
process of checking that the data they entered is correct and
complete before it gets saved or processed.

Examples of validation:
  - Making sure the Name field is not left blank
  - Making sure the Email field looks like a real email address
  - Making sure the user picked an option from a dropdown

ASP.NET MVC has built-in tools to handle all of this
automatically using something called Data Annotations --
labels you add to your Model that define the rules.


========================================================
WHAT WE ARE BUILDING
========================================================

An RSVP form for an event with 4 fields:
  - Name
  - Email
  - Phone
  - Will you attend? (Yes/No dropdown)

When the form is submitted:
  - If all fields are valid  --> the user sees a "Thanks" page
  - If any field is invalid  --> the form reloads with error messages

STEP 2 -- CREATE THE MODEL (GuestResponse.cs)
========================================================

WHAT IS A MODEL?
A Model is a C# class that defines the shape of your data.
Here, it defines what information the form collects.

We will also add Data Annotations -- these are special labels
(called "attributes") written above each property that tell
ASP.NET what rules to enforce when validating the form.

WHERE TO ADD IT:
  Right-click the "Models" folder --> Add --> Class
  Name it: GuestResponse.cs

PASTE THIS CODE (basic version, no validation yet):
----------------------------------------------------------
using System;
using System.Collections.Generic;
using System.Linq;
using System.Web;

namespace WebApplication7.Models
{
    public class GuestResponse
    {
        public string Name       { get; set; }
        public string Email      { get; set; }
        public string Phone      { get; set; }
        public bool?  WillAttend { get; set; }
    }
}
----------------------------------------------------------

WHAT IS bool? (with a question mark)?
  A regular bool can only be true or false.
  A bool? (nullable bool) can be true, false, OR null (empty).
  We use bool? here because when the form first loads,
  the user hasn't picked Yes or No yet -- so it starts as null.

----------------------------------------------
ADDING VALIDATION RULES TO THE MODEL
----------------------------------------------

To add validation, we import System.ComponentModel.DataAnnotations
and place attributes above each field.

UPDATED GuestResponse.cs (with validation):
----------------------------------------------------------
using System;
using System.Collections.Generic;
using System.Linq;
using System.Web;
using System.ComponentModel.DataAnnotations; // Required for validation

namespace WebApplication7.Models
{
    public class GuestResponse
    {
        [Required(ErrorMessage = "Please enter your name")]
        public string Name { get; set; }

        [Required(ErrorMessage = "Please enter your email address")]
        [RegularExpression(".+\\@.+\\..+",
            ErrorMessage = "Please enter a valid email address")]
        public string Email { get; set; }

        [Required(ErrorMessage = "Please enter your phone number")]
        public string Phone { get; set; }

        [Required(ErrorMessage = "Please specify whether you'll attend")]
        public bool? WillAttend { get; set; }
    }
}
----------------------------------------------------------

WHAT EACH ATTRIBUTE DOES:

  [Required(ErrorMessage = "...")]
    --> Makes the field mandatory. If the user leaves it
        blank and submits, the error message is shown.

  [RegularExpression(".+\\@.+\\..+", ErrorMessage = "...")]
    --> Checks that the email matches the pattern:
        something @ something . something
        (e.g., user@example.com)
        If it doesn't match, the error message is shown.

IMPORTANT: These rules are checked automatically by ASP.NET
when the form is submitted. You do not need to write
any manual checking code.


========================================================
STEP 3 -- CREATE THE CONTROLLER (HomeController.cs)
========================================================

WHAT DOES THE CONTROLLER DO HERE?
The controller manages TWO versions of the form page:

  GET request  --> User visits the page, show the blank form
  POST request --> User submits the form, check if it's valid

WHERE TO ADD IT:
  Right-click "Controllers" --> Add --> Controller
  --> MVC 5 Controller - Empty --> Name it: HomeController

PASTE THIS CODE:
----------------------------------------------------------
using System;
using System.Collections.Generic;
using System.Linq;
using System.Web;
using System.Web.Mvc;
using WebApplication7.Models; // Connects your model to this controller

namespace WebApplication7.Controllers
{
    public class HomeController : Controller
    {
        // Handles: /Home/Index
        // Just shows the home/welcome page
        public ActionResult Index()
        {
            return View();
        }

        // Handles: GET /Home/RsvpForm
        // Shows the blank form when the user first visits the page
        [HttpGet]
        public ActionResult RsvpForm()
        {
            return View();
        }

        // Handles: POST /Home/RsvpForm
        // Runs when the user clicks "Submit RSVP"
        [HttpPost]
        public ActionResult RsvpForm(GuestResponse guestResponse)
        {
            // ModelState.IsValid checks all the [Required] and
            // [RegularExpression] rules defined in GuestResponse.cs
            if (ModelState.IsValid)
            {
                // All fields passed validation -- go to Thanks page
                // and send the guestResponse data with it
                return View("Thanks", guestResponse);
            }
            else
            {
                // Something failed validation -- reload the form
                // ASP.NET will automatically remember what the user
                // typed and show the relevant error messages
                return View();
            }
        }
    }
}
----------------------------------------------------------

KEY CONCEPTS:

  [HttpGet]
    --> This action only runs when the user navigates TO the page.
        It shows a fresh, empty form.

  [HttpPost]
    --> This action only runs when the user SUBMITS the form.
        ASP.NET automatically maps the submitted form fields
        to the GuestResponse object passed as a parameter.

  ModelState.IsValid
    --> ASP.NET automatically checks every validation rule
        defined in the model. If all rules pass, this is true.
        If any rule fails, this is false.

  return View("Thanks", guestResponse)
    --> Goes to the Thanks.cshtml view AND passes the
        guestResponse data so Thanks.cshtml can display it.

----------------------------------------------
VIEW 2: RsvpForm.cshtml (The Form)
----------------------------------------------

WHERE TO ADD IT:
  Right-click the [HttpGet] RsvpForm() method --> Add View
  View name: RsvpForm
  Template: Empty (without model)

PASTE THIS CODE:
----------------------------------------------------------
@model WebApplication7.Models.GuestResponse

@{
    ViewBag.Title = "RsvpForm";
}

<h2>RSVP Now!</h2>

<html>
<head>
    <link href="~/Content/Stylr.css" rel="stylesheet" type="text/css" />
</head>
<body>
    <div>
        @using (Html.BeginForm())
        {
            <p>Your Name:  @Html.TextBoxFor(x => x.Name)</p>
            <p>Your Email: @Html.TextBoxFor(x => x.Email)</p>
            <p>Your Phone: @Html.TextBoxFor(x => x.Phone)</p>

            <p>
                Will You Attend?
                @Html.DropDownListFor(x => x.WillAttend, new[] {
                    new SelectListItem() { Text = "Yes, I'll be there", Value = bool.TrueString },
                    new SelectListItem() { Text = "No, I cannot come",  Value = bool.FalseString }
                }, "Choose an option")
            </p>

            @Html.ValidationSummary()

            <input type="submit" value="Submit RSVP" />
        }
    </div>
</body>
</html>
----------------------------------------------------------

HOW EACH PART WORKS:

  @model WebApplication7.Models.GuestResponse
    --> Tells the View it will work with GuestResponse data.
        This is required so the helper methods like
        TextBoxFor and DropDownListFor can reference the fields.

  @using (Html.BeginForm())
    --> Creates an HTML <form> tag that submits back to the
        same URL (RsvpForm) using POST. The "using" block
        automatically closes the form tag at the end.

  @Html.TextBoxFor(x => x.Name)
    --> Creates a text input box linked to the Name field.
        The "x =>" is a lambda expression meaning "for this
        model, use the Name property."

  @Html.DropDownListFor(x => x.WillAttend, ...)
    --> Creates a dropdown linked to the WillAttend field.
        The third argument "Choose an option" is the default
        blank option shown before the user picks one.

  @Html.ValidationSummary()
    --> This is where error messages appear when the form
        fails validation. ASP.NET automatically collects all
        error messages from the model's attributes and
        displays them here as a list.

  <link href="~/Content/Stylr.css" ...>
    --> Links to a CSS file for styling the form.
        The "~/" means "start from the root of the project."

========================================================
HOW VALIDATION FLOWS (END-TO-END SUMMARY)
========================================================

1. User visits /Home/RsvpForm
     --> [HttpGet] RsvpForm() runs --> blank form is shown

2. User fills in the form and clicks "Submit RSVP"
     --> Form data is sent via POST to /Home/RsvpForm

3. [HttpPost] RsvpForm(GuestResponse guestResponse) runs
     --> ASP.NET automatically fills guestResponse with
         the data from the form fields

4. ASP.NET checks every validation rule in GuestResponse.cs:
     - Is Name filled in?
     - Is Email filled in AND in the right format?
     - Is Phone filled in?
     - Did the user pick Yes or No from the dropdown?

5a. IF all rules pass (ModelState.IsValid == true):
     --> User is sent to Thanks.cshtml with their data

5b. IF any rule fails (ModelState.IsValid == false):
     --> RsvpForm.cshtml reloads with the user's previous
         input still in the fields, and error messages
         appear where @Html.ValidationSummary() is placed

========================================================
QUICK REFERENCE: VALIDATION ATTRIBUTES
========================================================

  ATTRIBUTE                          WHAT IT CHECKS
  ---------------------------------  -----------------------------------
  [Required]                         Field must not be empty
  [Required(ErrorMessage = "...")]   Same, with a custom error message
  [RegularExpression("pattern")]     Value must match a regex pattern
  [StringLength(max)]                Value cannot exceed max characters
  [Range(min, max)]                  Number must be between min and max
  [EmailAddress]                     Value must look like an email

  IN THE VIEW:
  @Html.ValidationSummary()          Shows all error messages in a list
  @Html.ValidationMessageFor(x=>x.Field)  Shows error for one specific field

  IN THE CONTROLLER:
  ModelState.IsValid                 True if all rules passed, else false
